
## Context
Devices publish a fixed binary MQTT payload (big-endian) to topic:
  telecom/{tenant_id}/{site_id}/{device_id}/telemetry

Source of truth: MQTT Telecom RMS Generic Hex Packet Specification Updated v1.2 Complete.

### Packet version
- Latest only: v1.2 / packet_version 0x03 / 425 bytes (0x1A9).
- Older 188-byte (v1.1) payloads are not accepted.

### Encoding
- Big-endian
- Invalid sentinels: U8=0xFF, U16=0xFFFF, I16=0x7FFF, U32=0xFFFFFFFF → null in model
- Voltage / current / temp / %x10 / Hz / L/h: value ÷ 10
- Battery charge/discharge kW: value ÷ 100
- Power W / energy Wh / minutes / hours / counts: as-is

### CRC
- CRC-16/MODBUS (init 0xFFFF, poly 0xA001 reflected) — verified against PDF example 0x1DB4 / 0x9A1A
- Base CRC at 0x9E over bytes 0x00–0x9D → Crc16 / IsCrcValid
- Extension CRC at 0x1A7 over bytes 0xA0–0x1A6 → ExtensionCrc16 / IsExtensionCrcValid
- Note: PDF table lists extension_crc at 0x16B; example and length 0x1A9 place it at 0x1A7.
  Reserved buffer is 0x167–0x1A6 (skip; fill 0x00).
