
## Context
Devices publish a fixed 160-byte binary payload (big-endian) to MQTT topic:
  telecom/{tenant_id}/{site_id}/{device_id}/telemetry


### 2. TelemetryPacket.cs — decoded model
A C# record or class with nullable fields for every field in the payload.
Null means the device sent the invalid sentinel (0xFF / 0xFFFF / 0x7FFF / 0xFFFFFFFF).

  string TenantId, SiteId, DeviceId

---
Rules:
- Validate payload length == 160, throw ArgumentException if not
- All reads are big-endian (manually shift bytes — do NOT use BinaryPrimitives which is little-endian by default on x86)
- Use private helpers: ReadU8, ReadU16, ReadI16, ReadU32
- Apply null sentinels per field spec above
- Validate CRC16 using the standard CRC-16/ARC or CRC-16/IBM polynomial (0x8005), compute over bytes[0..157], compare to bytes[158..159]
- Set a bool IsCrcValid on the packet
- Parse topic string to extract TenantId, SiteId, DeviceId
